    Accepted Template Params
    • Engine_t --> another class...THE main driver for the Grid...contains the shapes, points, (and basically everything about the Grid)
    • The main 'Grid' class acts as a global container for all the various grid Engines...the functions described below have are identified by their Engine_t name
  • Inside the Grid classes are hidden all sorts of optimizations to save both memory space AND speed. The memory space is managed by minizing the amount of stored data points in the grid. For instance a Cube grid needs only ONE Vector of data going from min--max. The entire 3D cube can be expressed as combinations of elements from this list. The acsess to the points of the Grid is maintained by extraction operators.
  • The UniformGrid is the most up-to-date and functional. The others are less so (becuase they are quite specialized for each shape furthermore the UniformGrid has been choosen as the Master Shape Grid Base Engine (see 'Shape')
    Grid Engines...The Main Driver sets
    • NullGrid
      • Nothing, has nothing, is nothing but a name and a place for empty sets
    • UniformGrid
      • A Rectangular 3D grid. The uniform implies a constant step size between each point...each direction (x,y,z) can have a different step size
    • SurfCubeGrid
      • Creates a cube in 3D that is NOT filled, only the surface is populated with points
    • FullCubeGrid
      • creates a simple filled cube. All steps sizes are the same along each x,y,z driection
    • SurfSphereGrid
      • Creats a Surface Sphere (NOT FILLED) in polar corrdinates. (If you wish a sphere in Cartesian coords use the 'Shape' class)
    • FullSphereGrid
      • Creats a Filled Sphere in polar corrdinates. (If you wish a sphere in Cartesian coords use the 'Shape' class)
!!NOTE!!
See the 'Looping' Example for important Info.

Function:
Grid(coord<> &min, coord<> &max, coord<int,3> &dim)
IMPORTANT:
The grids Engine_t handling
UniformGrid-->uses all pts
[(xmin, ymin, zmin), (xmax, ymax, zmax), (dimx, dimy, dimz)]
SurfSphereGrid-->(only the first 'dims' entry will be used from 'dims', both 'Rmin' and 'Rmax' will be set to max[0])
[(Rmax, minTheta, minPhi), (Rmax, maxTheta, maxPhi), (Dim, Dim, Dim)]
FullSphereGrid-->(only the first 'dims' entry will be used from 'dims')
[(Rmin, minTheta, minPhi), (Rmax, maxTheta, maxPhi), (Dim, Dim, Dim)]
Input:
min--> the (x,y,z) minima
max--> the (x,y,z) maxima
dim--> number of divisions along each direction
Description:
creates a Grid ASSUMING that the Engine_t can take the inputs (coord<> &min, coord<> &max, coord<int,3> &dim)
Example Usage:
coord<> mins(-1,-1,-1), maxs(1,1,1);
coord<int, 3> dims(10,10,10);
Grid<UniformGrid> myG(mins, maxs, dims); //a ready to be used grid
Function:
Grid(double min, double max, int dim)
IMPORTANT:
Engine_t Handleing

SurfCubeGrid-->uses all pts
[min=xmin, ymin, zmin; max=xmax, ymax, zmax; dim=xdim, ydim, zdim]
FullCubeGrid-->uses all pts
[min=xmin, ymin, zmin; max=xmax, ymax, zmax; dim=xdim, ydim, zdim]
SurfSphereGrid-->uses all pts
[min=Max Theta...min theta assumed to be 0; max=Max Phi..min phi assumed to be 0; dim=theta dim, phi dim; R is assumed to be 1]
FullSphereGrid-->uses all pts
[min=min R; max=max R; Theta assumed to go from 0--Pi; Phi assumed to go from 0--2Pi; dim=Rdim, Theta dim, Phi dim]
Input:
min-->a single minimum value
max-->a single maximum value
dim-->a single dimension
Description:
creates a grid object assuming all 3 (x,y,z) directions have the same min, max, and dim
Example Usage:
Grid<SurfSphereGrid> sg(Pi/2, Pi/4, 10);
Grid<UniformGrid> ug(1.,2.,3); //WILL FAIL

Function:
void Scale(coord<> mulBy);
void Scale(double xmul, double ymul, double zmul);
IMPORTANT:
Supported Engines...

UniformGrid-->full support
SurfCubeGrid-->full support
FullCubeGrid-->full support
FullSphereGrid-->Uses ONLY the first entry (can only scale 'R')
SurfSphereGrid-->Uses ONLY the first entry (can only scale 'R')
Input:
mulBy--> a coord with the values (xmul, ymul, zmul)
xmove--> amount to move in X direction
ymove--> amount to move in the Y direction
zmove-->amount to move in the Z direction
Description:
recalculates the Grid upon a a scaling factor..IF the the multipliers are negative then the inverse of the numbers are taken (multipliers can really only be positive)
Example Usage:
Grid<UniformGrid> myG(mins, maxs, dims);
myG.Scale(4,4,4); //grid is now 4 times the size it was at the start (same dims however)
myG.Scale(-2,-2,-2); //shrink the total grid by 1/2 (same dims however)
