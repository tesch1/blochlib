template<class Type>
class Vector : public MemChunk<Type>{....
Accepted Template Params
Type --> another class...typically a number type (int, doulbe, complex, float....)...but can be anything (default=NONE)
MemChunk is a class that handles the memory allocation and deallocation for ALL the various containers here
Notes:: mathematical operations are ELEMENT BY ELEMENT except for a few like 'norm,' 'dot,' 'max, ' and 'min.'

Matrix--Vector operations are described in 'Matrix'

!!NOTE!!
NUMERICAL NOTE::DO NOT confuse this class with the STL 'vector' (the one here has a CAPITAL 'V'). The STL vector class is much better suited as a 'holder' or an object that simply holds a bunch of object...this class is meant for performing numerical operations on large lists...it does maintain the same resizing/adding/dropping options as the STL version, but they are a bit slower then the STL version

NUMERICAL NOTE:: The default compiling options DOES NOT CHECK ANY BOUNDS...if you go outside the Vectors length you will crash the program...this is for SPEED...if it does not have to check the bounds then it is one less operation it must do. YOU CAN TURN THIS BACK ON by defining 'VBoundCheck 1' in the compile string
i.e. g++ -DVBoundCheck myprog.cc

Function:
Vector<T> fftshift(const Vector<T> &in)
Vector<T> ifftshift(const Vector<T> &in)


Input:
Vector<T> --> a vector of any type
Description:
Simply 'rotates' the Vector....places the last half of the vector as the first half, to achieve the 'spectral' ordering seen in most spectra. ifftshift returns the vector to the normal (algorithmic) ordering before on should perform an ifft.
Example Usage:
Vector<double> myV(5, 2);
Vector<complex> myFFT=fftshift(fft(myV));
//this should be the same as 'myV'
Vector<double> myIFFT=Re(ifft(ifftshift(myV)));

Function:
template<class Func>
void apply( Func &f)

template<class Func>
void apply( Func &f, const Range &r)

Input:
f--> a class with the T operator()(T in, int i) defined for it.

r--> the row range object

Description:
applys a funciton on the data inside the matrix...the class 'Func' MUST have this operator

T operator()(T in, int i)

where 'in' is the vector data element, and i is the index defined in order for the apply to work.

If the Ranges are specified then only those elements in the vector specified in the ranges will be effected.
If no ranges are specified all the element will be effected.
Example Usage:
class myF{
myF(){}
double T operator()(double in, int i){
return (i+1);
}

Vector<double> fm(2,5); //fm={5,5}
myF func;
fm.apply(func); //fm={1,2}

Function:
void pop()
IMPORTANT:
if the Vector is very long this opperation becomes VERY expensive. Because the list is not a linked list, but a continous array, the droppin process must copy all the old data excet for the last one
Input:
none
Description:
resizes the vector to a length length-1...
it preserves all the data and just dropping the LAST element in the list
this is equivilent to 'resizeAndPreserve(size()-1)'
Example Usage:
Vector<complex> tmpVec(3, 2);
tmpVec.pop(); //new vector length is 2 with elements 1--2 are all still 2
Function:
void push_back(T num)
IMPORTANT:
if the Vector is very long this opperation becomes VERY expensive. Because the list is not a linked list, but a continous array, the adding process must copy all the old data before adding the new element

Input:
num --> Adds the element 'num' to the back of the list and increases the vector length by one
Description:
resizes the vector to a length length+1...
this is like 'resizeAndPreserve(size()+1, num)'
Example Usage:
Vector<complex> tmpVec(3, 2);
tmpVec.push_back(complex(0,3)); //new vector length is 4 with elements 1--3 are all still 2 the 4th elemenet is complex(0,3)
Function:
void resize(int newL)
void resize(int newL, T num)
IMPORTANT:
NOTE::data integrity is NOT garenteeded after this operations (most likely all exsisting data will be lost)

Input:
newL --> the new size of the vector
num --> the vector will be filled with 'num' after resizing
Description:
resizes the vector to a length newL...

Example Usage:

tmpVec.resize(5); //new vector length is 5
tmpVec.resize(5, 4); //new vector length is 5 filled with 4's
Function:
void resizeAndPreserve(int newL)
void resizeAndPreserve(int newL, T num)
IMPORTANT:
NUMERICAL NOTE:: this is slower then 'resize' becuase of the data copying
Input:
newL --> the new size of the vector

Description:
resizes the vector to a length newL...
if newL < old length then ALL the data up to newL is kept...
if newL > old Length then all data is kept, but the NEW entries are NOT initiallized (or are filled with 'num')

Example Usage:
Vector<complex> tmpVec(3, 2);
tmpVec.resizeAndPreserve(5); //new vector length is 5 with elements 1--3 are all still 2
tmpVec.resizeAndPreserve(5, complex(0,3)); //new vector length is 5 with elements 1--3 are all still 2 and 4-5 are complex(0,3)
